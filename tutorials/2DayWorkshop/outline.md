# ServiceX 2D Workshop Outline

ServiceX 2D is a curriculum for a 2 day workshop to help people move their analysis to ServiceX.

**Workshop Mission:** Get users using ServiceX in their analysis

**Workshop Goals:**
1. **Sell It** - Introduce what ServiceX is and why to use it
2. **Teach It** - Get users generically aquatinted with ServiceX
3. **Enable It** – Enable users to incorporate ServiceX fully into their specific analysis

**Target Audience:** People who are not currently using ServiceX and have limited exposure to it.

## Curriculum Outline:

### Day 1:

#### 09:00 - (1) - ServiceX Introduction Presentation

- Should help the users understand the basic architecture
- Should cover the long term benefits
- Why should the students spend the next 2 days learning how to use this tool!

#### 09:30 - (2) - 15 Minute Histogram Challenge

- https://tryservicex.org/challenge/

#### 09:45 - (2) - 45 Minute Analysis Challenge

- https://tryservicex.org/challenge/
- During this time it is important to make sure that each student is using the correct backend either uproot or func_adl

#### 10:30 - Break

#### 11:00 - (2) - Analysis Characterization and Workflow Selection

- During this time we should lead the students through a characterization of their analysis
- This should allow them to look at the list of the workflows that we have create and choose a couple to learn.
- They will then complete the workflows and learn them while asking questions
- This should get them thinking about how they can implement these tools into their analysis, but not actually implement them yet.
- Each student should select 3 or 4 workflows to work on.

#### 12:00 - Lunch

#### 13:00 - (1) - get_structure() Demonstration

- Some students will have done this in the last section, but this is a cool tool we should be showing everyone!
- This is a great way to continue to build our sales pitch.
- Students should then try to use get_structure() with their own dataset

#### 13:30 - (2) - Backend Breakout Sessions

##### Uproot Breakout Session

##### Func_ADL Breakout Session

- Some users will need to use FuncADL. This means we should be ready to give them some extra help on getting it built into their analysis.
- First we will give a short presentation on the structure of FuncADL Queries
- Then we will give a tutorial on building up FuncADL Queries

##### TopCP Breakout Session
